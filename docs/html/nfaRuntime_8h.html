Returns the index of regex which matches the most of the input from a buffer.

Characters are read from the buffer until either the buffer is exhausted, a non-printable character is encountered, or all of the regexes have given up on matching the text.

Parameters
nfa_listThe array of GrrEngine regex objects.
numThe length of the array.
sourceThe buffer holding the text. It does not need to be null-terminated.
sizeThe number of characters to be processed.
processedPointer to where the number of processed characters is stored.
scoreIf a regex matched some portion of the text, then this points to where the most number of characters matched is stored.
Returns
The index of the regex with the longest match of the input or -1 if either no such match was found or any of the parameters were NULL/zero.

Determines if a string contains a substring which matches the regex.

The function will stop processing characters when either it exhausts the specified length, a newline is encountered, or a non-printable character is encountered with tolerant set to false. If tolerant is true, then the function will skip over such characters and treat them as beginning and end of lines. For example, if the regex is "^a+" and the string is "aa\x00aaad", then both sequences of "a"'s will be considered matches. However, they will be considered separate matches and so *end-*start will be 3.

Parameters
nfaThe GrrEngine regex object.
stringThe string (does not have to be null-terminated).
lenThe length of the string.
startA pointer which will, if not NULL, point to the index of the beginning of the longest match if one was found.
endA pointer which will, if not NULL, point to the index of the character after the end of the longest match if one was found.
cursorA pointer which will, if not NULL, point to the index of the character where the function stopped searching.
tolerantIndicates whether or not the function should exit upon encountering a non-printable character.
Returns
GRR_RET_OK if a substring match was found. GRR_RET_BAD_ARGS if either nfa or string is NULL. GRR_RET_NOT_FOUND if no substring match was found. GRR_RET_BAD_DATA if the string contained a non-printable character and tolerant was set to false.
